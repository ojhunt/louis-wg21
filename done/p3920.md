# Summary

In a joint EWG and LEWG session in Kona, there was consensus to remove the trivial relocation feature ([P2786R13][])
from C++26 due to various issues that won't be repeated here, with a desire to fix these issues in the C++29
time frame. This paper contains the wording for the removal, based on top of the current working draft.


# Effect on Trivial Relocation NB comments

## Resolve the following NB Comments as accepted

**Revert P2786:**

- ✅ [US 73-131][] 20.2.2, 20.2.6 Cleaning up the trivial relocation APIs

## Resolve the following NB comments as moot

**Respecify using memcpy rules:**

- ✅ [CN 6.9.1p9][] `[basic.types.general]` Trivial relocatability should imply bitwise relocatability
- ✅ [US 46-085][] 11.2, 20.2.6 Make trivial relocation implementable by `memcpy`

**Fix the special tokens:**

- ✅ [US 44-082][] 11.1 `[class.pre]` Conditional trivially relocatable types

- ✅ [US 10-023][] 5.11, 11.01, 11.2, A.9, C.1.4, C.1.6 Remove `*_if_eligible` keywords

**Support more types:**

- ✅ [CA-136][] 20.2.6p16 `[obj.lifetime]` Allow const-qualified types for `relocate`
- ✅ [CA-137][] 20.2.6p9 `[obj.lifetime]` Allow const-qualified types for `trivially_relocate`

**Fix implicit property:**

- ✅ [US 47-084][] 11.2p2 `[class.prop]` Implicitly deleted move operation should not disable trivial relocation CWG3049

## Undo changes in [class.prop]{.sref}

::: rm
> [2]{.pnum} A class C is _default-movable_ if
>
> - [2.1]{.pnum} overload resolution for direct-initializing an object of type `C` from an xvalue of type `C` selects a constructor that is a direct member of `C` and is neither user-provided nor deleted,
> - [2.2]{.pnum} overload resolution for assigning to an lvalue of type `C` from an xvalue of type `C` selects an assignment operator function that is a direct member of `C and is neither user-provided nor deleted, and
> - [2.3]{.pnum} `C` has a destructor that is neither user-provided nor deleted.
>
> [3]{.pnum} A class is _eligible for trivial relocation_ unless it
>
> - [3.1]{.pnum} has any virtual base classes,
> - [3.2]{.pnum} has a base class that is not a trivially relocatable class,
> - [3.3]{.pnum} has a non-static data member of an object type that is not of a trivially relocatable type, or
> - [3.4]{.pnum} has a deleted destructor,
> except that it is implementation-defined whether an otherwise-eligible union having one or more subobjects of polymorphic class type is eligible for trivial relocation.
>
> [4]{.pnum} A class `C` is a _trivially relocatable class_ if it is eligible for trivial relocation and
>
> - [4.1]{.pnum} has the `trivially_relocatable_if_eligible` _class-property-specifier_,
> - [4.2]{.pnum} is a union with no user-declared special member functions, or
> - [4.3]{.pnum} is default-movable.
>
> [5]{.pnum} [A class with const-qualified or reference non-static data members can be trivially relocatable.]{.note}

> [6]{.pnum} A class `C` is _eligible for replacement_ unless
>
> - [6.1]{.pnum} it has a base class that is not a replaceable class,
> - [6.2]{.pnum} it has a non-static data member that is not of a replaceable type,
> - [6.3]{.pnum} overload resolution fails or selects a deleted constructor when direct-initializing an object of type `C` from an xvalue of type `C` ([dcl.init.general]{.sref .unnumbered}),
> - [6.4]{.pnum} overload resolution fails or selects a deleted assignment operator function when assigning to an lvalue of type `C` from an xvalue of type `C` ([expr.assign]{.sref .unnumbered}, [over.assign]{.sref .unnumbered}), or
> - [6.5]{.pnum} it has a deleted destructor.

> [7]{.pnum} A class `C` is a _replaceable class_ if it is eligible for replacement and
>
> - [7.1]{.pnum} has the `replaceable_if_eligible` _class-property-specifier_,
> - [7.2]{.pnum} is a union with no user-declared special member functions, or
> - [7.3]{.pnum} is default-movable.

> [8]{.pnum} [Accessibility of the special member functions is not considered when establishing trivial relocatability or replaceability.]{.note}

> [9]{.pnum} [Not all trivially copyable classes are trivially relocatable or replaceable.]{.note}

## Undo changes in [obj.lifetime]{.sref}

::: rm
> ```cpp
> template<class T>
>   T* trivially_relocate(T* first, T* last, T* result);
> ```
>
> > [9]{.pnum} _Mandates_: `is_trivially_relocatable_v<T> && !is_const_v<T>` is `true`. `T` is not an array of unknown bound.
> >
> > [10]{.pnum} _Preconditions_:
> >
> > - [10.1]{.pnum} `[first, last)` is a valid range.
> > - [10.2]{.pnum} `[result, result + (last - first))` denotes a region of storage that is a subset of the region reachable through result ([basic.compound]{.sref .unnumbered}) and suitably aligned for the type `T`.
> > - [10.3]{.pnum} No element in the range `[first, last)` is a potentially-overlapping subobject.
> >
> > [11]{.pnum} _Postconditions_:
> > No effect if `result == first` is `true`. Otherwise, the range denoted by `[result, result + (last - first))` contains
> > objects (including subobjects) whose lifetime has begun and whose object representations are the original object
> > representations of the corresponding objects in the source range `[first, last)` except for any parts of the object
> > representations used by the implementation to represent type information ([intro.object]{.sref .unnumbered}). If any
> > of the objects has union type, its active member is the same as that of the corresponding object in the source range.
> > If any of the aforementioned objects has a non-static data member of reference type, that reference refers to the same
> > entity as does the corresponding reference in the source range. The lifetimes of the original objects in the source
> > range have ended.
> >
> > [12]{.pnum} _Returns_: `result + (last - first)`.
> >
> > [13]{.pnum} _Throws_: Nothing.
> >
> > [14]{.pnum} _Complexity_: Linear in the length of the source range.
> >
> > [15]{.pnum} _Remarks_: The destination region of storage is considered reused ([basic.life]{.sref .unnumbered}). No constructors or destructors are invoked.
> >
> > [Overlapping ranges are supported.]{.note}
>
> ```cpp
> template<class T>
>   constexpr T* relocate(T* first, T* last, T* result);
> ```
>
> > [16]{.pnum} _Mandates_: `is_nothrow_relocatable_v<T> && !is_const_v<T>` is `true`. `T` is not an array of unknown bound.
> >
> > [17]{.pnum} _Preconditions_:
> >
> > - [17.1]{.pnum} `[first, last)` is a valid range.
> > - [17.2]{.pnum} `[result, result + (last - first))` denotes a region of storage that is a subset of the region reachable through result ([basic.compound]{.sref .unnumbered}) and suitably aligned for the type `T`.
> > - [17.3]{.pnum} No element in the range `[first, last)` is a potentially-overlapping subobject.
> >
> > [18]{.pnum} _Effects_:
> >
> > - [18.1]{.pnum} If `result == first` is `true`, no effect;
> > - [18.2]{.pnum} otherwise, if not called during constant evaluation and `is_trivially_relocatable_v<T>` is `true`, then
> >   has effects equivalent to: `trivially_relocate(first, last, result);`
> > - [18.3]{.pnum} otherwise, for each integer `i` in `[0, last - first)`,
> >   - [18.3.1]{.pnum} if `T` is an array type, equivalent to: `relocate(begin(first[i]), end(first[i]), *start_lifetime_as<T>(result + i));`
> >   - [18.3.2]{.pnum} otherwise, equivalent to: `construct_at(result + i, std​::​move(first[i])); destroy_at(first + i);`
> >
> > [19]{.pnum} _Returns_: `result + (last - first)`.
> >
> > [20]{.pnum} _Throws_: Nothing.
> >
> > [Overlapping ranges are supported.]{.note}

:::
